# .canonical_allowlist.txt — pre-push canonical-state ratchet allowlist
#
# Lines listed here are exempted from the pre-push gate that blocks new
# inline `WHERE status / WHERE created_at / WHERE sent_at / invoice_date
# / logged_at` filters outside src/core/. Use this only as an escape
# valve when a PR can't fully migrate a callsite to canonical_state in
# the same change (e.g. the migration is large enough to need its own PR).
#
# ## Format
#   - One full source line per entry, exactly as it appears in the diff
#     (without the leading `+` from git diff output).
#   - Comments with `#` are ignored.
#   - Blank lines are ignored.
#
# ## Maintenance rule (RATCHET ONLY)
#   - Adding entries here is OK, but document why in the PR description.
#   - The allowlist must shrink, not grow, over the migration arc. Each
#     PR-2..PR-6 should remove entries as it migrates the matching
#     consumer. PR-6 (cleanup) should leave this file at zero entries
#     and the ratchet becomes a hard ban.
#
# ## When to skip the gate entirely
#   `CANONICAL_STATE_SKIP=1 make ship` — only for chore/revert/docs
#   pushes where you're confident no new ad-hoc filter slipped in.
#
